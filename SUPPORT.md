# Support

`frame-by-frame` is currently in early development and has not published a usable package.

## Where to ask

- Use the bug report form for reproducible defects in committed behavior.
- Use the feature proposal form for API, capability, or documentation proposals.
- Use an issue for a project question only when the answer would improve public documentation for other contributors.

GitHub Discussions may be enabled later when the project has enough usage to benefit from a dedicated community support channel.

## What not to post publicly

- Report vulnerabilities through the process in [SECURITY.md](SECURITY.md).
- Report conduct concerns through the private channel in [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).
- Never include credentials, access tokens, private media, or personal data in an issue or reproduction.

## Scope of support

Maintainers will prioritize actionable reports that fit the documented project scope. The project does not currently provide guaranteed response times, private implementation consulting, or support for unreleased APIs.
