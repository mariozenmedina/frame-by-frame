# Security Policy

## Current support status

`frame-by-frame` has not published its first release. There is no stable version or compatibility commitment yet, but security reports affecting the repository or unreleased core are welcome.

After releases begin, this document will list the versions receiving security updates.

## Reporting a vulnerability

Do not open a public issue for a suspected vulnerability.

Use GitHub's private vulnerability reporting for this repository:

1. Open the repository's **Security** tab.
2. Select **Report a vulnerability**.
3. Describe the affected code or behavior, realistic impact, reproduction steps, and any suggested mitigation.

Please avoid including secrets, personal data, or destructive proof-of-concept material beyond what is necessary to understand the report.

The maintainers will assess the report, communicate through the private advisory, and coordinate disclosure when appropriate. Response times may vary while the project is maintained by a small team; no fixed service-level agreement is currently offered.

## Conduct reports

While the project is small, the same private reporting feature is also the confidential channel for Code of Conduct reports. Prefix the report summary with `[Code of Conduct]` so it can be handled under the correct policy.
