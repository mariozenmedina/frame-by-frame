Pin `1.0.0-rc.1` exactly when reproducible candidate testing matters. Do not use an unqualified `latest` install for this prerelease.

## What ships

- deterministic pixel and normalized-progress timelines;
- forward, reverse, gap, multi-clip, and global or per-segment easing behavior;
- independent vertical, horizontal, and simultaneous scroll axes;
- native video rendering with bounded latest-value-wins seeking;
- native, full-file, manual, first-use, and viewport loading policies with aggregate readiness;
- ordered responsive replacements and reduced-motion behavior;
- opt-in video-backed 2D canvas rendering;
- SSR-safe ESM entries, public TypeScript declarations, and no runtime dependencies.

## Package entries

| Entry                         | Contract                                                       |
| ----------------------------- | -------------------------------------------------------------- |
| `@frame-by-frame/core`        | Recommended video-only controller, timeline, errors, and types |
| `@frame-by-frame/core/video`  | Explicit alias for the root video-only API                     |
| `@frame-by-frame/core/canvas` | Opt-in factory supporting native-video and canvas bindings     |
| `@frame-by-frame/core/types`  | Runtime-empty, type-only entry                                 |

The package is ESM-only. Supported Node.js lines are `^22.18.0` and `^24.11.0`; browser consumers need an ESM-capable build pipeline.

## Compatibility evidence

The repository's deterministic gates cover behavior, types, coverage, SSR imports, package entries, installation from the locally packed tarball, bundle budgets, documentation, dependency review, and CodeQL. Real-browser evidence remains platform-qualified:

- the recorded Windows Professional 25H2 x64 run passed all 14 Chromium and 14 Firefox scenarios plus nine WebKit scenarios; five WebKit media-presentation cases were explicit Windows-only skips;
- issue [#25](https://github.com/mariozenmedina/frame-by-frame/issues/25) contains a reported 14-of-14 WebKit run on macOS 14.6.1 Intel using Playwright's frozen macOS 14 WebKit build; its documentation contribution remains open;
- issue [#27](https://github.com/mariozenmedina/frame-by-frame/issues/27) requests supplementary Linux evidence and does not block this candidate.

Read the exact [browser validation record](../browser-validation-results.md) and [browser-support guide](../guides/browser-support.md). These results do not imply support for every Safari release, WebKit build, operating system, architecture, codec, asset host, or device.

## Known limitations

- Native video exposes approximate media times from browser events. `requestVideoFrameCallback()` improves observation where available, but the package does not guarantee frame-exact presentation across decoders.
- Codec selection, CORS, byte-range delivery, preload cost, memory pressure, and representative production assets remain integration responsibilities.
- Canvas rendering uses a video decoder and 2D copy; it is not a WebCodecs, WebGL, or WebGPU pipeline.
- Framework adapters are not included. Vue will be the first maintained example after the initial core release; other frameworks remain open to community proposals.
